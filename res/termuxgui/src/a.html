You can set up a graphical environment on termux as follows. For local testing purposes, download vnc client and x11 server android applications, we'll connect this to local host. First download the x11-repo

pkg install x11-repo

Let's start with a vnc server, which is probably the easiest.

pkg install tigervnc

You should get /data/data/com.termux/files/usr/bin/vncserver, which you can use to start the server using vncserver -localhost. An initial run will prompt you for credential setup, so just take not of what you enter there. The usual vnc server port is 5900+N, which is within the range you can use in Termux's userland OS space. If everything is good, you should see the following output

New 'localhost:1 ()' desktop is localhost:1

Starting applications specified in /data/data/com.termux/files/home/.vnc/xstartup
Log file is /data/data/com.termux/files/home/.vnc/localhost:1.log

So now the vnc server is running on localhost:1, and you can try it out with the android vnc client application. What you'll end up with is the VNC client connected to an empty screen. Also, add export DISPLAY=":1" to you bash .profile. Next we need to add a window manager, and we'll use Fluxbox for this. 

pkg install fluxbox

To get it to start with the VNC server, edit ~/.vnc/xstartup as follows

#!/data/data/com.termux/files/usr/bin/sh
## Fluxbox desktop.

# Generate menu.
fluxbox-generate_menu

# Start fluxbox.
fluxbox &

Killing the vncserver and restarting it should set everything in motion, and upon reconnecting the VNC client you should be presented with the window manager

You can ofcourse also connect from a different machine, but a local VNC client is good for testing. With this setup you can develop GTK, Qt, SDL, etc applications from your phone. This helps a lot if you want to continue with projects and don't want to carry along an entire machine just for sitting down and coding in a coffee shop or while traveling. Just pull from git and continue on. If you take a look in the repositories you'll see some pretty sophisticated stuff already ported, and you'll find that you can compile some useful stuff from source that will actually run

Alternatively you can download noVNC and use that as a client which will allow you to use any web browser as a client, simply pointing it to 192.168.2.251:6080/vnc.html?host=192.168.2.251&port=6080 for example.
