The Range library is an STL extension that makes iterators and algorithms more powerful by making them composable. C++ 20 has the ranges library, but if you're using an older standard, you can use ericniebler/range-v3.

git clone https://github.com/ericniebler/range-v3
cd ./range-v3
mkdir ./build
cd ./build
cmake -DCMAKE_INSTALL_PREFIX:PATH=/usr/bin ../
make -j8
make install

Its a header only library, and the tests will take fairly long to compile. The library headers should live at /usr/include/range.

The three main components of Ranges are Views, Actions and Algorithms. Views and Actions use a pipe syntax like rng|adapt1|adap2|....

Normally to manipulate collections we use STL and its algorithms. for example

#include<algorithm>
#include<iostream>
#include<vector>
int main(int argc,char** argv){
	std::vector<int> a={0,1,2,3};
	std::vector<int> b;
	std::copy_if(
		a.begin(),
		a.end(),
		std::back_inserter(b),
		[](int c){
			return n%2==0;
		}
	);
	std::vector<int> c;
	std::transform(
		b.begin(),
		b.end(),
		std::back_inserter(c),
		[](int d){
			return d*2;
		}
	);
	for(auto d:c){
		std::cout<<d<<" ";
	}
	std::cout<<std::endl;
	return 0;
}

The above sets up a std::vector, performs a filter for even numbers using std::copy_if, then doubles these using std::transform. Among other things, Ranges simplifies stuff like this. Using Ranges, you can perform the above as follows

auto b=
	a|
	ranges::view::filter(
		[](int c){
			return n%2==0;
		}
	)|
	ranges::view::transform(
		[](int d){
			return d*2;
		}
	)
;

In the above you'll see that there are not iterators. Using Ranges you work with a level of abstraction, so iterators are not visible in the interface.

A view is a lightweight wrapper that presents a view of the underlying sequence of elements in a custom way without mutating or copying it. Views use non-owning reference semantics. An adaptor returns a view, for example auto a=ranges:;views::reverse(v) has adaptor ranges::views:;reverse returning a view.

If you have a C++20 compiler you can include ranges. In the below example we use ericniebler/range-v3 as built above

#include<iostream>
#include<range/v3/all.hpp>
#include<vector>
int main(int argc,char** argv){
	std::vector<int> a={1,2,3,4,5,6};
	for(auto b:a){
		std::cout<<b<<" ";
	}
	std::cout<<std::endl;
	auto b=a
		|ranges::views::filter([](int n){return n%2==0;})
		|ranges::views::transform([](int n){return n*2;})
	;
	for(auto c:b){
		std::cout<<c<<" ";
	}
	std::cout<<std::endl;
	return 0;
}

take

ranges::views::take takes the first N elements and discards the rest

#include<iostream>
#include<vector>
#include<range/v3/all.hpp>
int main(int argc,char** argv){
  auto a=ranges::views::ints(1,ranges::unreachable)|ranges::views::take(10);
  for(auto b:a){
    std::cout<<b<<" ";
  }
  std::cout<<std::endl;
  return 0;
}

Also take not eof ranges::views::ints which generates a range for you up to ranges::unreachable which is infinity. You can also use ranges::views::ints(N) to create an infinite range of elements with the value of N. Piping ranges::views::ints to ranges::views::take(N) selects N elements from the sequence generated.

X_of

ranges::all_ov, ranges::any_of and ranges::none_of can be used to perform tests on a collection based on a lambda or functor's boolean return value.

#include<range/v3/all.hpp>
#include<iostream>
#include<vector>
int main(int argc,char **argv){
  std::vector<int> v={0,1,2,3,4,5};
  std::cout<<ranges::any_of(v,[](int i){return i==5})<<std::endl;';
  std::cout<<ranges::all_of(v,[](int i){return i==5})<<std::endl;';
  std::cout<<ranges::non_of(v,[](int i){return i==5})<<std::endl;';
  return 0;
}

zip

Let's look at how we can zip two sequences. Zipping takes two sequences and combines them into one. Using ranges::views::zip_with you can provide a lambda or functor taking two arguments that defines how the two input sequences will be zipped.

#include<iostream>
#include<vector>
#include<range/v3/all.hpp>
int main(int argc,char** argv){
  auto a=ranges::views::ints(1,ranges::unreachable)|ranges::views::take(10);
  auto b=a|ranges::views::transform([](int c){return c*2;});
  auto c=ranges::views::zip_with([](int d,int e){return std::make_pair(d,e);},a,b);
  std::cout<<ranges::views::all(a)<<'\n';
  std::cout<<ranges::views::all(b)<<'\n';
  for(auto d:c){
    std::cout<<"["<<d.first<<":"<<d.second<<"]";
  }
  std::cout<<std::endl;
  return 0;
}

cartesian_product

ranges::views::cartesian_product constructs a view over a set of collections that returns all possible permutations, for example for {0,1}, {0,1}, and {0,1 the permutations are {(0,0,0),(0,0,1),(0,1,0)...(1,1,1)}. It constructs this in a lazy manner.

#include<iostream>
#include<vector>
#include<range/v3/all.hpp>
int main(int argc,char** argv){
  std::vector<std::string> a={"a","b","c"};
  std::vector<std::string> b={"c","d","e"};
  std::vector<std::string> c={"f","g","h"};
  auto d=ranges::view::cartesian_product(a,b,c);
  for(auto e:d){
    std::cout
      <<"["
      <<std::get<0>(e)<<":"
      <<std::get<1>(e)<<":"
      <<std::get<2>(e)
      <<"]"
      <<std::endl
    ;
  }
  return 0;
}

partition_copy

ranges::partition_copy can be used to partition a collection as illustrated in the following example. Based on the bool returned by the lambda or functor the collection will be split into two.

#include<iostream>
#include<vector>
#include<range/v3/all.hpp>

     ranges::views::ints(1,ranges::unreachable)
    |ranges::views::take(10);
  std::vector<int> b;
  std::vector<int> c;
  ranges::partition_copy(
    a,
    ranges::back_inserter(b),
    ranges::back_inserter(c),
    [](int d){return d%2==0;}
  );
  std::cout<<ranges::views::all(a)<<std::endl;
  std::cout<<ranges::views::all(b)<<std::endl;
  std::cout<<ranges::views::all(c)<<std::endl;
  return 0;
}

group_by

ranges::views::group_by groups contiguous elements based on a binary predicate, on other words for a range, adjacent elements are compared in some way, and if a if a chunk or segment of the range has all its adjacencies such that the predicate evaluates to true, they will be grouped together

#include<iostream>
#include<vector>
#include<range/v3/all.hpp>
int main(int argc,char** argv){
  auto a=
   ranges::views::ints(1,ranges::unreachable)
  |ranges::views::transform([](int i){return 2*i+(i%2);})
  |ranges::views::take(16);
  std::cout<<(ranges::views::all(a))<<std::endl;
  std::cout<<"----------------------------------------"<<std::endl;
  for(auto b:
    (
     a
    |ranges::views::group_by([](auto b,auto c){
        return b%3==c%3;
    })
  )){
    std::cout<<(ranges::views::all(b))<<std::endl;
  }
  return 0;
}

The above generates the following output

[3,4,7,8,11,12,15,16,19,20,23,24,27,28,31,32]
----------------------------------------
[3]
[4,7]
[8,11]
[12,15]
[16,19]
[20,23]
[24,27]
[28,31]
[32]

Taking a look at the above, using predicate b%3==c%3, comparing 3 and 4 you'll find that 3%3 is 0 and 4%3 is 1, which means these are two seperate group. Taking a look at 4 and 7 you'll find that 4%3 is 1 and 7%3 is also 1, so these are grouped together. Taking a look at 7 and 8 you'll find that 7%3 is 1 and 8%3 is 2, so these are two seperate groups. The comparison and grouping is performed throughout the range in order to deliver the results above.

You can also use this to do frequency counts on characters, as illustrated in the following example

#include<iostream>

  auto a=std::string("lorem ipsum sit consecutar");
  std::cout<<a<<std::endl;
  std::cout<<"----------------------------------------"<<std::endl;
  ranges::sort(a);
  for(auto b:a|ranges::views::group_by([](char d,char e){
    return d==e;
  })){
    std::cout<<(ranges::views::all(b))<<std::endl;
  };
  std::cout<<"----------------------------------------"<<std::endl;
  return 0;
}
